## 7. Build / Validation Notes

Important operational note from recent work:

- `dotnet build` spawns many `.NET Host` processes on this machine.
- The user is sensitive to unnecessary build/test runs because of that behavior.
- Prefer static inspection while editing.
- Only run minimal targeted builds when needed.
- After any build verification, explicitly clean up build servers if required.

Recommended validation sequence for the next session:

1. Build only the directly affected project first.
2. If that succeeds, build `src/CsArchViewer.Avalonia/CsArchViewer.Avalonia.csproj`.
3. Only then consider solution-level verification.
4. If background build hosts remain, shut down build servers before ending the session.
